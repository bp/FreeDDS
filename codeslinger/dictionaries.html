In its most basic form, a dictionary is a text file that describes data. That file contains a series of definitions. Nothing more. Common definitions include the names of data axes, the size of each axis, sample increments and origins, and data units. More detailed definitions describing trace formats and sample types may also be present.

Dictionaries are also used for parfiles and command line parameters.

Although DDS formally uses 20 or more predefined dictionaries, Only 3 stand out for their frequency of use and the need for both users and developers to understand them.

  • Data Dictionaries - Typically a text file dictionary is associated with each dataset. It defines characteristics of the binary data. For some predefined formats (e.g. segy) DDS uses the line header as a binary dictionary. A complete dictionary is created by combining binary line header information with the predefined segy format.

  • Command Line Dictionaries - Input/output filenames and other command line parameters used by an application are stored in the cmd: dictionary.

  • Parfile Dictionaries - Parfiles contain additional parameters that are either too long or too difficult to put on the command line. They are an extension of the command line. Parameter definitions are put into a file and that file is specified on the command line as par=file. These definitions are stored by DDS in the par: dictionary.
