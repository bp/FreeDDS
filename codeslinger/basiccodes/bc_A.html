Purpose:   Introduction to a bare minimum of the DDS API

Code Outline:

  • Open a file for access to samples
  • Retrieve data characteristics needed for the task (n1, n2, ..., d1, d2, ...)
  • Allocate an array to hold data
  • Read through the data accumulating a sum of samples
  • Report the result
  • Clean up before exiting

DDS resources:

  • fddx_in / cddx_in - open input (drop headers, pass samples)
  • fdds_scanf / cdds_scanf - scan definitions
  • fdds_axis_prod / cdds_axis_prod - product of axis sizes
  • fdds_malloc8 / cdds_malloc - allocate memory (needed only for f77)
  • fddx_read / cddx_read - read data
  • fdds_free / cdds_free - free memory (needed only for f77)
  • fdds_close / cdds_close - close dictionary and associated binary
