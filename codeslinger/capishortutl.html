There are relatively few DDS functions that are required for typical application development. These few functions can be adapted to 95% of application needs. All the other functions are available for special needs as they arise, but if you only learn functions on the short list, you should be able to perform almost any task and greatly simplfy your coding experience.

Sort by Function

    Mission Critical
    • cddx_in - open input (drop headers, pass samples)
    • cddx_out - open output
    • cddx_read - read data
    • cddx_write - write data
    • cdds_dict - set dictionary mode
    • cdds_scanf - scan definitions
    • cdds_close - close dictionary and associated binary
    Other Heavy Lifters
    • cdds_malloc - allocate memory
    • cdds_lseek8 - trace seek
    • cdds_free - free memory
    • cdds_scank - scan definitions for keys
    • cdds_printf - print definitions
    • cddx_dict - set dictionary mode
    • cdds_axis_prod - product of axis sizes
    • cdds_scant - scan definitions, retrieve a token
    Debugging and Communication
    • cdds_openpr - open print file
    • cdds_closepr - close print file , exit program
    • cdds_prtcon - print to print file and console
    • cdds_prtmsg - print to print file
    • cdds_prterr - print error to console and print file
    • cdds_errors - return # errors
    Headers Too
    • cddx_in2 - open input (pass headers and samples)
    • cdds_index - query field index
    • cdds_member - query member tag
    • cdds_getxx - get a field
    • cdds_putxx - put a field
    Parallel Utilities
    • cdds_initmpi - initialize MPI
    • cdds_initopenmp - initialize OpenMP
