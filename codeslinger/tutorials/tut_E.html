Purpose:   Work with data, ignore but don't change trace headers. Include error detection features.

Task:   Write a program that reads header-attached traces and applies a taper at the beginnng and end of non-zero samples. Check for problems during setup. Report problems as they occur. Before real work starts, check if any problems were found and halt the process if needed.

Outline:

  • Open the print file, detect request for help message
  • Open a file for access to samples and headers
  • Check for errors
  • Retrieve data characteristics needed for the task (n1, n2, ..., d1, d2, ...)
  • Retrieve user parameters
  • Open the output dictionary and binary stream
  • Check for errors
  • Allocate arrays to hold input data and work space
  • Check for errors in setup, stop if any were found
  • Read the data, apply the taper, periodically report progress
  • Write the output
  • Clean up and exit

New DDS resources:

  • fddx_in2 / cddx_in2 - open input (pass headers and samples)
  • fdds_errors / cdds_errors - report # calls to [f,c]dds_prterr()

Examples
