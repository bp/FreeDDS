Examples progress from basic to advanced. Later examples build on knowledge gained in earlier ones. Each example code shows how the different functions can be performed. The first few use the DDS API in as simple a form as is practical. Starting with example E, codes start to use more robust programming methods suitable to production applications.

  1. Access to data samples    (program avg)
    • Open a binary file (or steam) for input
    • Retrieve data attributes
    • Allocate an array for data
    • Read through the data one trace at a time
    • Clean up DDS resources

  2. Access to user parameters    (program scale)
    • Read parameters from command line and parfile
    • Open a binary file (or steam) for output
    • Read 2D planes of data
    • Write 2D planes of the result

  3. Communicate with users    (program tttn)
    • Use DDS print files to log information
    • Acknowledge users request for runtime help
    • Use subroutines or functions for parameter retrieval and algorithm setup

  4. Process changes data sizes    (program transp)
    • Adjust output dictionary to reflect process changes in data attributes
    • Deal with data of unknown rank

  5. Working around data headers    (program taper)
    • Process data samples, pass trace headers unchanged
    • Use error detection features

  6. Process changes type of sample    (program rcfft)
    • Use FFT wrappers to convert REAL samples to COMPLEX
    • Adjust output dictionary to reflect new sample type

  7. Working with trace headers    (programs xyrange, hdr_apply)
    • Extract selected headers from input traces for analysis or use
    • Give users the flexibility of defining the headers to grab

  8. Dynamically change trace headers (under development)
    • Extend trace structure to add application-specific headers
    • Completely redefine traces to include only a few headers
    • Allow users to easily define header mapping relationships
