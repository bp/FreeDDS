A

alias
A list of possible names for a definition, separated by spaces. Used by fdds_in, fdds_out, and the fdds_scan* functions.

B

bin, bin_tag
An integer used by DDS to identify a binary data stream.

bootstrap dict
par: and cmd: dictionaries glued together.

buf, in_buf, out_buf
A memory buffer. 'in_' and 'out_' are used to elaborate on direction of information/data transport.

D

data_alias
An alias used to define overrides for input and output binary data file names.
E.g.   'in= in_dict   in_data= file.segy'   on a command line will cause in_dict to use file.segy as the binary data file, regardless of the any other file specified within in_dict.

DATA_PATH
A variable used to override the path to any data files written. DATA_PATH=/dat22/users/workspace will cause all binary data files to be written to /dat22/users/workspace, even when the command is executed from a different location and out= points to a different location. The path can be overridden by specifying a specific full path name with the out_data (or similar) alias.

DATA_SUFFIX
A suffix automatically appended to an output alias (e.g. out=) name to distinguish a dictionary from the binary data it refers to. This can be overridden by using out_data= bin_name. The current default value for DATA_SUFFIX is "_@" but can be changed by specifying a different suffix value.

DDS type
A type of variable known to DDS. E.g. DDS_INTEGER, DDS_REAL, or DDS_COMPLEX. Used by cdds_index, and fdds_index to describe the type of value for which the index is used.

definition ['name= value']
The 'name= part of the definition cannot contain white space and always ends with '='. The value part of a defintion can be zero or more tokens separated by white space. value can span many lines. A definition ends when a new name= is found. For example, 'axis= t x y' is a common definition.

dict, dictionary
A collection of definitions. If it is a 'data dictionary', the definitions describe a binary data set. If it is a 'command line' or 'parfile' dictionary, definitions are used to control the behavior of the application.

F

field
A member of a SEQUENCE. Think of SEQUENCE as a structure that describes each data trace within a dataset. If SEQUENCE is a segy trace, it is a structure containing many trace headers, followed by the Samples. Any single header is a field and the segy 'Samples' field is an array containing many values.

field_genus
The genus of a field (E.g. float, int, complex).

field_name
The name associated with a single field. E.g. 'Samples', or 'SrcX'.

field_tag
An identifier of a specific field with a Sequence.

flags
xx

fmt
A character string describing format conversion attributes to apply to a list of other arguments. Used by fdds_scanf, fdds_printf, fdds_prtcon, and friends. It is the same format as used by the "C" compiler language.

format_alias
An alias used to define overrides for input and output binary data. E.g.   'in= in_dict  in_format= fcubex'   on a command line will cause DDS to treat data associated with in_dict as fcubex data, regardless of the format specified within in_dict.

G

genus
The general class of a data object. E.g. 'integer', 'float', and 'complex' are genus values that do not imply precision.
