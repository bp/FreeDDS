Setup the following environmental variables:
  • DDSROOT - set to the root DDS directory (eg. /usr/local/DDS.2003-07-01). This is used by all the make files in identifying DDS subdirectories during the making of libraries and applications.

  • DDS_DIST - set to the distribution directory (eg. ${FreeUSP}/dist). This is used by the application installation make file (MakeRules), and installation scripts (ddsbincopy, ddsexecopy & ddsmancopy). It is the base for all production scripts, binaries and man pages. The distribution directory can be located anywhere but probably should be the same as used by FreeUSP. You will need to make sure that ${DDS_DIST}/bin is in your path in order to execute the binaries.

NOTE: DDS should already be setup to make for the following systems. We've also used Sun, SGI, PGI, HP and Intel compilers. DO NOT use older gf77 compilers because they do not support array pointers. The new gfortran works if you user version >= 4.2.2. If a specific system needs to be added or modified, see Configuration Changes section.

Optional Runtime Monitoring:
  • DDS_MONITOR - This is set in dds-set-path to point to a directory where log files can be put for keeping track of application usage. If it points to a valid directory, then a log file will be created for each application which keeps track of successful executions of each application program. When a program terminates successfully the "Monitor" routine in c_print.c in src/lib/dds directory checks for the environmental variable, "DDS_MONITOR" (which gets set by dds-set-path), and if the "${DDS_MONITOR}/prog.log" log files exists then a single line will be added indicating the binary path used, date, time, user, time of execution, and basic command line parameters. Any time a log file exceeds 10000 lines, it is replaced with the last 1000 lines.
