DDS provides a set of default mappings between various data formats. Default mappings are adequate for most, but not all, field conversions. The end user should understand what the defaults are, and override them as needed.

Below are tables of default header mappings. These tables only include the explicit header mappings between headers of different names. All headers with identical names between formats are implicitly mapped by default. (NOTE: header "Spare.Hdr.RecNum" is not the same as "RecNum).

The segy format reserves 60 bytes for spare Trace Header fields and segy1 format reserves 24 bytes for sapre trace header fields. Usage of these fields varies widely within the industry. Thus, they may be used differently within a particular segy dataset. Even "standard" header definitions are often ignored by vendors requiring the necessity of overriding the default mappings.

Also, be aware that some fields may be ambiguous in some formats because they contain the same information as another field in the same format. As such, no default mapping is made INTO any field that is contained in any union.
