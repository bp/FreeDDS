While each definition "value" contains every character from the "=" character to the beginning of the next definition (including any and all "white space"), the definition "value" is further broken down into "tokens" which exclude are separated by "white space" (with the exception of any "white space" that is either escaped with "\" or is included inside double quotes. The following table has several examples showing the "tokens" for the definition foo. 

    
     DEFINITION          TOKEN1   TOKEN2    TOKEN3
     ------------------  -------  -------   -------
 1)  foo= caca           "caca"      -         -
 2)  foo=caca widget     "caca"   "widget"     -
 3)  foo= caca widget    "caca"   "widget"  "thing"        
          thing 
 4)  foo=bar=caca...        -        -         -
 5)  foo= "tkn 1" caca   "tkn 1"  "caca"    "this & that"
      "this & that"

Once again, be careful with the command shell interacting with the special characters. For example, if you use double quotes to define tokens with white space or "=" characters, the command shell will strip off the quotes, leaving the tokens to be split up by the "white space" into more tokens. You can use single quotes outside the double quotes or more easily you can put the definitions in a Par File.
