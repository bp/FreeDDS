"fmt:format=" binary Definitions are used to describe the structure of binary data (see Fmt Control). The Definition names are always prefixed by "fmt:" followed by the actual "format" name. Binary definitions are kept in the Dictionary associated with binary data. DDS binary structures are the fundamental unit for data transfer and buffer mapping. An example for USP is: 

fmt:usp= 
   typedef USP_LINE PREAMBLE;
   typedef USP_TRACE SEQUENCE;

fmt:USP_LINE= typedef align(1) struct {
      GREEN_PREFIX line_green_prefix;
      USP_CHAR EqpCod, CrwNam[6], DatTyp, DgTrkS[2], PrcNam[10],
         PrcNam_GAP[4], PrcDat[8], OACLin[8], JobNum[8];
      ...
      USP_SHORT HlhEnt, HlhByt;
      unsigned1 HlhBuf[USP_HLH_SIZE];
      unsigned1 usp_line_pad[USP_LINE_PAD];
   } USP_LINE;

fmt:USP_TRACE= typedef align(1) struct {
      GREEN_PREFIX green_prefix;
      USP_SHORT SGRNum, SGRAmp;
      USP_INT SGRDat;
      USP_SHORT FlReFN, FlStNm, PREPIn, InStUn, InStAp, PREPRc, RcStUn,
         RcStAp, ToStAp, ToTmAA, ToStUn, ToTmAU, SrComp, RcComp,
         SrStat, RcStat, TrHdID, SrRcAz;
      ...
      SAMPLE_TYPE Samples[axis_size(1)];
      unsigned1 usp_trace_pad[USP_TRACE_PAD];
   } USP_TRACE;

These binary Definitions also describe how to derive one DDS binary format from another, given an old and new format (or alias name; see I/O Control for alias names). Default "fmt:" Definitions are kept in special dictionaries for each format named "DDS_PATH/format.fmt.dds". All DDS binary Definitions are kept current in the data Dictionary. This provides a comprehensive history of binary data formats. An example of a "fmt:" conversion Definition is: 

fmt:usp:segy.SEGY_TRACE= ...
