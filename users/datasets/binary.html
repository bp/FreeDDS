Several Data Dictionary Definitions are used to describe binary data. Their values are retrieved, when binary data is opened by an application program. The "data=" Definition defines where the binary filename is located. The "format=" Definition defines the binary format name. And the "fmt:format=" Definitions are used to define the actual binary format structure -- whether it's on disk or in an internal format where a specific host-dependent application code can manipulate the data fields.

Filenames:

By default, output binary data filenames are the same name as it's Dictionary suffixed by "_@" (eg. line7_@) or by the environment, DATA_SUFFIX (See environment). The binary file is also created by default in the current working or by the environment, DATA_PATH. This allows the binary data to be diverted to a special file system that may provide special capacity, speed, or backup support. This provides for simpler management of the smaller dictionaries in working project directories.
