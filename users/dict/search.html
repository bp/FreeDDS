Search Order:

When DDS searches (scans) for a definition, it can have multiple Dictionaries open for scanning. The first Dictionary listed for scanning will be scanned first. If the definition is not found then DDS will continue scanning the next Dictionary.

Global Data Dictionary Search:

  • cmd: - Dictionary contains the command line parameter definitions
  • par: - Dictionary contains any definitions specified by a "par=" file in addtions the the "cmd:" definitions.
  • ${CWD}/project.dds - Project dictionary for setupng up project-related definitions, for example "DATA_PATH=".
  • ${HOME}/.ddsrc - User dictionary to setup up user personal preferences, for example specifying "PRINT_PATH=".
  • data dictionaries contain all previous historical definitions as well as the "par:" definitions.

Local Parameter Search:

DDS programs can search "par:" for local parameters associated with the current program execution. During opening of an input or output dataset, programs search for parameter definitions in the data Dictionary thus having access to "global" parameters from previous steps in the processing history.

Extended Dictionaries:

Extended dictionaries are used to find some definitions. These include DDS_PATH, DATA_PATH, DATA_SUFFIX and device attributes. Extended dictionaries are searched until the desired definition is found. Each level broadens the scope from problem specific to a wider arena. The following table shows the order by which DDS does these searches: 
 Dictionary   Purpose (precedence)
 ----------   -------------------------------
 env:         environment defaults (lowest)
              (not used for DATA_PATH & DATA_SUFFIX)
 data_dict    problem specific
 par:         command line defaults (highest)
