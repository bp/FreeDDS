Special dictionaries are required for run time support to define the binary formats (see Formats) and header field mappings (see Mapping)). They are builtin to the DDS system as defaults, but any of them can be modified to reflect local conventions for header mapping or special binary formats. (They reside in "${DDSROOT}/src/lib/dds3.) For DDS to be informed of a modification, place the modified file within a directory named by DDS_PATH (see Runtime Control). These custom dictionaries are used, if found; otherwise the internal defaults installed within DDS are used. These dictionaries are summarized by the following table. 

Dictionary         Description
-------------      -------------------------------------
fmt.fmt.dds        format definitions
segy.fmt.dds       segy format (& su)
segy1.fmt.dds      segy1 format
usp.fmt.dds        usp format
asp.fmt.dds        Generic DDS (asp, sep & cube formats)

fmt.map.dds        field mappings
segy.map.dds       map to segy (& su)
segy1.map.dds      map to segy1
usp.map.dds        map to usp
asp.map.dds        map to Generic DDS (asp, sep & cubes)
