The segy binary header, SEGY_BINARY, contains a sequence of fields taking up 400 bytes in length. The field values, associated with the binary data, are also available as Dictionary Definitions by preceding their names with "segy_" (eg. segy_LineNum=). They may be examined in the Dictionary by end users or retrieved by applicationprograms. End users may override any of the field values associated with binary input data. For example: 

comment= override binary header value
segy_NumTrc= 48
segy_SmpInt= 8000
segy_NumTrc= 1048

The first two above could just as well have been specified by their corresponding axis Definitions (assuming the default "axis= t x y"): 

size.x= 48
delta.axis(1)=8.0
units.axis(1)=usec

The default fields that make up the SEGY_BINARY structure are defined in "${DDSROOT}/src/lib/dds3/segy.fmt.dds" and may be modified by editing that file and putting it into a directory defined by "${DDS_PATH}". These defaults are summarized by their nominal offset, name, type, and element count.
