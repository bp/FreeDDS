If you need to modify the EBCDIC header of a segy file, simply define segy_card3200 (for either segy or segy1 format). This definition is defined as up to 40 tokens, each one up to 80 characters. The following example can be used to convert a file to a segy1 format dataset and specify the EBCDIC header (the single quotes (') are only needed when used on the command line and are not needed in a par file): 

bridge in=dataset out=out.data out_format=segy1 \
'segy_card3200= "line1 First Line" "line2 \"quote\" something" line3\=nothing line4'

Running ddscan on the output file and dumping the lineheader will give: 

        --- Line Header: PREAMBLE (3200 bytes) ---

 -FIELD-   -OFST- -LEN- -BYTS-      -VALUE- -TYPE-
 card3200      0  3200   3200               ebcdic1 
      "line1 First Line                                                                "
      "line2 "quote" something                                                         "
      "line3\=nothing                                                                  "
      "line4"

The double quotes (") around "quote" must be escaped with "\" if they are to be part of the EBCDIC header. Also, any equal sign (=) that is intended to be part of the header must also be escaped; otherwise the equal symbol will terminate the segy_card3200 definition and start a new definition.
