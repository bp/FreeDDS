At times you may need to map header values based on the ordinal values of any given axis. The ordinal values go from 0 to size for any given axis. For example if you want to set TrcNum and RecNum to go from 1 to the size of the 2nd and 3rd axes, you could use: 

edit:out= force.2 force.3
write:out.TrcNum= 1 + ordinal(2)
write:out.RecNum= 1 + ordinal(3)

NOTE1: In order to use the ordinal function, you need to use it only with "read:" or "write:" and not with any "map:" definitions because the ordinal does not make sense and is not defined until actually reading or writing traces.

NOTE2: When using the ordinal function, you may need to force it to be the expected values for any axis. This is apparently some quirk of DDS because the ordinal values are also used internally for padding traces. As such, you cannot both force the ordinal value and pad traces at the same time.
