Conversion fmt Definitions can also describe how to derive one DDS structure to another. This is distinct from derived format Definitions used to describe the structure of existing binary data. These Definitions reside in special dictionaries; normally they are not in the history associated with binary data. Most formats are descibed within the ${DDSROOT}/src/lib/dds3/format.fmt.dds Dictionaries.

A symbol table is built when opening binary data. The table is built from input fmt Definitions, if the data and Definitions already exist. If not, the table is built from these default conversion fmt Definitions in the appropriate ${DDSROOT}/src/lib/dds3/format.fmt.dds Dictionary. The Definitions, implied by the table, are uncompiled as derived Definitions into the data Dictionary that describes the binary data.

This conversion fmt Definitions are of the form: 

fmt:old:new.defn= expression
where
  • old is the old format or "*"
  • new is the new format
  • defn is the Definition searched for
  • expression is the typedef expression
For example: 
fmt:*:usp.USP_LINE= ...

fmt:*:out.SAMPLE_TYPE=typedef complex8 SAMPLE_TYPE;

If old is "*" (wildcard), the definition matches any old format name. The new format name must match the format associated with the type being created.
