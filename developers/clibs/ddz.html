Introduction

libddz extends DDS to allow flexible host-dependent sample type mapping in the form of convenience routines. Flexible host-dependent mapping is useful when double precision data are input and precision needs to be maintained, or when a program is designed to manipulate input samples, regardless of type, without risk of losing information.

Note that libddz maps between different host-dependent types of the same genus. It cannot map between types in different genuses. For example, a float can be mapped to a double, but cannot be mapped to a complex nor a complex double.

As with the other DDS convenience routines, the developer need not contend with DDS dictionaries or internal buffer binaries. If any errors occur, they will be handled by the regular cdds_prterr routine which keeps track of errors.

After opening a file for input with cddx_mapin, cddx_mapin2, cddx_hdtin, or cddx_hdtin2, all of the other cddx convenience routines can use the resulting bin tag as though the file had been opened with cddx_in, cddx_in2, etc.

Functions and their purpose

cddz_getdict

Get the name of a dictionary when only the BIN_TAG is known

cddz_hdtin

Open a DDS dataset for input passing samples mapped to the host-dependent type

cddz_hdtin2

Open a DDS dataset for input, passing headers and mapping samples to the host-dependent type

cddz_hdtype

Detect the host-dependent type of a field

cddz_intype

Interrogate a DDS dataset for input sample genus and host-dependent type

cddz_mapin

Open a DDS dataset for input, passing samples mapped to an application specific host-dependent type

cddz_mapin2

Open a DDS dataset for input, passing headers and mapping samples to an application specific host-dependent type

cddz_md5sum

Get the md5sum of a buffer

cddz_scann

Get the name of a definition found by cdds_scan*()

cddz_sysconf

Get system configuration information
