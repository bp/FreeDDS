cddz_intype - interrogate a DDS dataset for input sample genus and host-dependent type

SYNOPSIS

#include "cdds.h"
#include "cddz.h"

BIN_TAG cddz_intype(alias, value, title, genus, host_type)

const char *alias
const char *value
const char *title
const char **genus
const char **host_type

DESCRIPTION

Function "cddz_intype" enables flexible, host-dependent sample type mapping. It is used in conjunction with "cddz_mapin" or "cddz_mapin2" to open a file in 2 stages. Between the intype and mapin calls, the program can decide on the mapping to perform based on information returned in arguments genus and host_type. The BIN_TAG and argument alias should not be changed between intype and mapin calls.

After opening a file for input with "cddz_intype" the input dictionary will be left open for scanning. The open process will look for "alias" for the dictionary, "alias_data" for overriding the binary, and "alias_format" for overriding the format. The input parameter "value" is the default value (eg. "stdin:") used for a fallback in cases where "alias" is not specified by the user. The input parameter "title" should be a brief one line description of the application program. It will return a BIN_TAG associated with the external binary upon successful opening, a value of -2 if the dataset is not specified and no default value is given.

LIMITATIONS

Convenience routines have a limitation where the alias argument can only have a single alias name.

EXAMPLE

BIN_TAG    in_tag,buf_tag;
const char *title, *genus, *hdt_in, *hdt_mapto;
 ...
in_tag=cddz_intype("in","stdin:",title,&genus,&hdt_in);
if (!strcmp(genus,"integer")) hdt_mapto="long long";
if (!strcmp(genus,"float"))   hdt_mapto="double";
if (!strcmp(genus,"complex")) hdt_mapto="complex double";
buf_tag=cddz_mapin("in","stdin:",title,&hdt_mapto,&in_tag);
 ...
