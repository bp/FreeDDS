Application libraries built within the Makefile

The DDS make structure allows a developer to create and maintain any desired application library within the Makefile. Simply define the set of source files as with any application. However, define the library name with "LibName=libname", and define a project name with "PrjName=prjname". For example, with the following Makefile (assuming all the source code is in the same directory): 

LibName:= sort
PrjName:= sort_utils
FSrcs  := sort.F 
CSrcs  := sort.c subs.c++
LibHdrs:= sort.h
.
.
.

you will create the "libsort.a" library with "make". Running "make install" will install that library under the directory "${DDSROOT}/lib/sort_utils/...". If any headers files are defined with "LibHdrs" then they will be installed under "${DDSROOT}/include/sort_utils".

NOTE: It is reccommended to make the name of the library (as well as the name of the source directory) start with "lib". If not, the Makefile will add "lib" to the name of the compiled library.

These libraries are installed under ${DDSROOT}/lib/${PrjName}/... to facilitate the creation of project-specific libraries that are separate from the distributed/stable DDS libraries. Since these libraries are installed in different directories from the other DDS libraries, applications that use them need to specify the library and project names. E.g. 

Libs   := sort
PrjLibs:= sort_utils
