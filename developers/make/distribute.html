Distribution of Code

Within BP use one of these two scripts to push out one or more installed programs and send a request to have them distributed to various locations under the production directory structure. A newer method is to use the "distribute" target of make:

  • Run "make distribute" to distribute all the installed files (startup script, man page & associated binaries).

  • Run "PushUpdate prog" to distribute your program, prog, including a man page, startup script, and all executable binaries for all platforms from the tstapps structure to the apps structure.

  • Run "PushUpdates" to distribute all of your program files (ie. files owned by $USER) that need to be updated from the tstapps structure to the apps structure.
