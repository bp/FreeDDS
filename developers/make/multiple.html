Multiple codes within same Makefile

The DDS make structure allows a developer to maintain multiple codes within the same directory and Makefile. Simply define the set of source files separately for each application or library by prefixing the application or library name onto the "FSrcs" and/or "CSrcs" definitions ("nameFSrcs" and/or "nameCSrcs"). Then when using "make" append "Name=name" onto the command line. For example, with the following Makefile (assuming all the source code is in the same directory): 

freqFSrcs  := freq.F t2w.F w2t.F

ddscanFSrcs:= ddscan.F
.
.
.

you could make the "ddscan" application with: "make Name=ddscan" and "make install Name=ddscan". Or with "make Name=freq" the "freq" program will be built.
