make Usage

These are the main make targets used for building DDS Applications. For a more complete list run "make help"" to get online help information on the various targets that can be used.

  • Run "make" to build the DDS application. If a startup script (with same name as the program name) does not exist then one will be generated for you.

  • Run "make install" to install the startup script into ${DDS_DIST}/bin; the manpage (same name as the program name with the ".1" extension) if one exists; and the compiled binary into it's appropriate path under ${DDS_DIST} corresponding to the current architecture. If no manpage exists, a template will be automatically generated. If the manppage is in the local working directory, then it will be moved to the "man1" subdirectory, That makes it easier to check the manpage with "man -M . <prog>.

  • Run "make clean" to remove local binary and objects files.

  • Run "make all" build the application on multiple machines (as defined by ${REMOTE_HOSTS} or determined at run-time) using "rsh". Because this mechanism simply uses "rsh" to do the remote makes on other machines, make sure your ".rhosts" file is setup correctly to do "rsh". Test this by running something like "rsh <machine> date". (You can override "rsh" with "ssh" by defining "DDSRSH=ssh".)

  • Run "make allinstall" install the application on multiple machines.

  • Run "make allclean" to remove local binaries and objects files for all machines.
