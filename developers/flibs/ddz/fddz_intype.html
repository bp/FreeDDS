fddz_intype - interrogate a DDS dataset for input sample genus and host-dependent type

SYNOPSIS

#include "fdds.h"
#include "fddz.h"

integer function fddz_intype(alias, value, title, genus, host_type)

character alias*(*)
character value*(*)
character title*(*)
character genus*(*)
character host_type*(*)

DESCRIPTION

Routine "fddz_intype" enables flexible, host-dependent sample type mapping. It is used in conjunction with "fddz_mapin" or "fddz_mapin2" to open a file in 2 stages. Between the intype and mapin calls, the program can decide on the mapping to perform based on information returned in arguments genus and host_type. The integer return value and argument alias should not be changed between intype and mapin calls.

After opening a file for input with "fddz_intype" the input dictionary will be left open for scanning. The open process will look for "alias" for the dictionary, "alias_data" for overriding the binary, and "alias_format" for overriding the format. The input parameter "value" is the default value (eg. "stdin:") used for a fallback in cases where "alias" is not specified by the user. The input parameter "title" should be a brief one line description of the application program. It will return a integer associated with the external binary upon successful opening, a value of -2 if the dataset is not specified and no default value is given, otherwise a value of -1 to signify an open failure.

LIMITATIONS

Convenience routines have a limitation where the alias argument can only have a single alias name.

EXAMPLE

integer   in_tag,buf_tag
character title*(*), genus*(*), hdt_in*(*), hdt_mapto*(*)
 ...
in_tag=fddz_intype('in','stdin:',title,genus,hdt_in)
if (index(genus,'integer').gt.0) hdt_mapto='long long'
if (index(genus,'float').gt.0)   hdt_mapto='double'
if (index(genus,'complex').gt.0) hdt_mapto='complex double'
buf_tag=fddz_mapin('in','stdin:',title,htd_mapto,in_tag)
 ...
