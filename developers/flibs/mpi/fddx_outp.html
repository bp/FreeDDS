fddx_outp - open an output file for parallel writing, mapping some compulsory and optional headers.

SYNOPSIS

#include <pfddx.h>

integer fddx_outp(alias, value, prog, in_bin, node, np, olddict, numcotags, numoptags, hw, tags)

character*(*) alias
character*(*) value
character*(*) prog
integer in_bin
integer node
integer np
character*(*) olddict
integer numcotags
integer numoptags
character*(DEFNNAME_MAX) hw(numcotags+numoptags)
integer tags(numcotags+numoptags)

DESCRIPTION

The function, "fddx_outp", opens an output file for parallel writing, mapping some compulsory and optional headers.
If the file is not opened succesfully, then an error is issued.
If the compulsory headers are not found, then an error is issued.
If the optional headers are not found, then a warning is issued.

INPUT

The parameter, "alias", specifies the parameter name defining the input file.

The parameter, "value", specifies the default filename.

The parameter, "prog", specifies the program name.

The parameter, "in_bin", specifies the input file binary tag.

The parameter, "node", specifies the MPI node number.

The parameter, "np", specifies the total number of MPI nodes.

The parameter, "olddict", specifies a dictionary from which to copy the history. This can be used to modify the output dictionary if, for example, the output has different dimensions to the input. If no modications are needed, then set this to ' '.

The parameter, "numcotags", specifies the number of compulsory trace header tags.
If the compulsory headers are not found, then an error is issued.

The parameter, "numoptags", specifies the number of optional trace header tags. If the optional headers are not found, then a warning is issued.

The parameter, "hw", specifies the array of compulsory headers, followed by the optional headers.

OUTPUT

The parameter, "tags", specifies the header tags (returned by cdds_member).

RETURN VALUE

fddx_outp returns the binary tag to the output file, if it was opened successfully, else returns -1.
