It is highly recommended to use "implicit none" which will greatly help to reduce typo's and programming errors.

fdds_openpr should be called to check for online help request or open the print file. The first time any API routine is called will cause DDS to initialize itself. This call will specifically initialize the print file; the user can always override and modify the print file creation via the PRINT_PATH environment. The '$Id: initialize.html 57 2009-02-11 23:09:37Z ehlersjw $' field will be filled in appropriately by RCS or CVS every time the file is checked out. The fdds_openpr routine will pull out what it needs, if available from the second parameter. Put '$Id: initialize.html 57 2009-02-11 23:09:37Z ehlersjw $' in a position that allows for maximum characters to be filled in to not overflow any card columns. 

       program filt
c
       implicit none
c
#include <fdds.h>
c     
      ier=fdds_openpr('filt', '$Id: initialize.html 57 2009-02-11 23:09:37Z ehlersjw $')
      if (ier.gt.0) call help()
